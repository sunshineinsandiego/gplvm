# Multimodal Shared Gaussian Process Latent Variable Model (Shared GPLVM)

Welcome to the **Shared GPLVM** repository. This project uses machine learning to find hidden connections between two very different types of data:
1.  **Medical Imaging:** High-dimensional ultrasound videos of patellar tendons.
2.  **Tabular Data:** Clinical health scores, biomechanics, and athlete game statistics.

Our goal is to learn a "Shared Latent Space"—a hidden map of patient health—that explains both the anatomy seen in the scan and the performance seen on the court.

---

## 1. The Intuition: Plato's Cave for Medicine

Imagine you are a doctor trying to understand the "True Health" of an athlete's knee. You cannot see "True Health" directly; it is an abstract concept. However, you can observe two things that result from it:
1.  **The Shadow on the Wall (Image):** You can take an ultrasound scan.
2.  **The Report Card (Data):** You can measure their pain levels and jump height.

A **Shared GPLVM** assumes that both the Image ($Y_1$) and the Data ($Y_2$) are generated by the same underlying, hidden variable ($X$).

If we can learn this hidden $X$, we can do magic tricks:
*   **Imputation:** If a patient has an ultrasound but forgot to fill out their pain survey, we can find their $X$ using the image, and then use $X$ to predict what their pain score *should* be.
*   **Discovery:** We can see if "structural damage" in the image actually correlates with "poor performance" in the stats, or if they are disconnected.

---

## 2. The Math (Simplified)

We model the relationship as:

$$ Y_{img} = f_1(X) + \text{noise} $$
$$ Y_{tab} = f_2(X) + \text{noise} $$

*   **$X$ (Latent Space):** A low-dimensional vector (e.g., 12 numbers) representing the patient's state. We assume $X$ follows a standard Bell Curve (Gaussian Prior).
*   **$f_1$ and $f_2$ (The Mappings):** These are functions that translate the hidden state $X$ into the observed data.
    *   $f_1$ translates "Health" $\to$ "Image Features".
    *   $f_2$ translates "Health" $\to$ "Clinical Scores".

Because the relationship between anatomy and pain is complex and non-linear, we use **Gaussian Processes (GPs)** to model $f_1$ and $f_2$. GPs are great because they don't just give a prediction; they give a prediction *plus uncertainty*.

---

## 3. Design Choices: Why did we build it this way?

### Q: Why "Two GPs" if it's a "Shared" model?
You might notice in the code (`src/shared_gplvm.py`) that we initialize `self.gp_img` and `self.gp_tab`.

**"I thought we wanted a shared model, not two different models?"**

We **do** have a shared model because they share the **Input ($X$)**.
However, we need two separate **Translators (GPs)** because the **Outputs ($Y$)** are completely different languages.

*   **GP 1 ($f_1$):** Needs to map $X$ to 768 dimensions of visual data (smooth, continuous pixel features).
*   **GP 2 ($f_2$):** Needs to map $X$ to 17 dimensions of clinical data (integers, scores, different scales).

If we used a single GP, we would be forcing the model to use the exact same mathematical "smoothness" (kernel lengthscale) to describe both pixel intensity and pain scores, which doesn't make sense. By splitting them, we allow the model to learn: *"The image changes slowly as health gets worse, but pain scores jump up suddenly."*

They remain "Shared" because during training, we force them to agree on a single $X$ for every patient.

### Q: Why Variational Inference?
Standard GPs scale poorly ($\mathcal{O}(N^3)$). If we have 10,000 video frames, a standard GP would crash the computer.
**Variational Inference** allows us to approximate the GP using a small set of "Inducing Points" (e.g., 50 anchor points). This makes the model fast and scalable to large datasets.

Crucially, Variational Inference allows us to handle **Missing Data**. If a patient is missing their Tabular data ($Y_2$), we don't have to throw them away. We just rely on the Image ($Y_1$) and the Prior ($X \sim \mathcal{N}(0,1)$) to figure out where they sit in the latent space.

## 4. Codebase Walkthrough

### 1. `src/compile_dataset.py` (The Data Engineer)
This script prepares the data. It:
*   Crawls the `scan/` directory for ultrasounds.
*   Reads `PPTTUS.csv` (clinical) and `game_stats.csv` (basketball).
*   **The "DPP" Step:** Ultrasound videos have hundreds of frames. We can't use them all. This script runs a "Diversity" algorithm (Determinantal Point Process) to pick the ~10 most unique/important frames from each video.
*   **Output:** It creates `data/manifest.csv`, a master list linking every image frame to its corresponding patient stats.

### 2. `src/shared_gplvm.py` (The Brain)
This file defines the model architecture.
*   **`SharedVariationalGPLVM`**: The container class. It holds the learnable Latent Embeddings ($X$) and the two GP sub-models.
*   **`LatentGP`**: A standard Variational GP module. We instantiate two of these: one for images, one for tabular data.

### 3. `src/train_gplvm.py` (The Teacher)
This script trains the model.
*   **`MultimodalDataset`**: A smart dataloader. It reads the CSV and loads images. Crucially, it runs the **DINOv3** vision model *on-the-fly* to convert images into feature vectors (embeddings).
*   **The Loop:** It optimizes the "ELBO" (Evidence Lower Bound). It tries to move the latent points $X$ around until they maximize the probability of observing the data we actually see.

### 4. `src/impute_and_evaluate.py` (The Test)
This script tests if the model actually learned anything useful.
*   **Reconstruction:** Can we recreate the data from the learned $X$?
*   **Inference Mode (`image_only`):** This is the "Magic Trick". It takes a new image, forgets everything else, and tries to find the best $X$ using *only* that image. Then, it projects that $X$ to predict the missing clinical scores.
*   **Attribution:** It calculates correlations to tell us: *"Latent Dimension 3 is highly correlated with Tendon Thickness."*

## 5. Tutorial: How to Run

### Step 1: Compile the Data
First, we need to link the images to the spreadsheets and select keyframes.

```bash
python src/compile_dataset.py \
  --scan_dir scan \
  --ppt_csv data/PPTTUS.csv \
  --cu_csv data/game_stats_CU.csv \
  --fd_csv data/game_stats_FD.csv \
  --out_manifest data/manifest.csv \
  --n_keyframes 10
```

### Step 2: Train the Model
Now we train the Shared GPLVM. This will learn the latent space $X$.

```bash
python src/train_gplvm.py \
  --manifest data/manifest.csv \
  --out_dir models/checkpoints \
  --epochs 500 \
  --latent_dim 10
```

### Step 3: Evaluate & Predict
Finally, we test the model. We can run it in `image_only` mode to simulate a scenario where we have the scan but no clinical data.

```bash
python src/impute_and_evaluate.py \
  --manifest data/manifest.csv \
  --model_dir models/checkpoints \
  --inference_mode image_only
```

**Output Interpretation:**
Look for the **Attribution Analysis** at the end of the output.
```
Latent Dimension 2:
  - V_SCORE: -0.85
  - AT_THICK: 0.72
```
This result would mean: "We found a hidden variable (Dim 2). When this variable goes up, Tendon Thickness goes up (0.72) and VISA Health Scores go down (-0.85). This is the 'Injury' dimension."
